Project Overview

Scalable Analytics with
Apache Spark

This project leverages the power of Apache Spark to perform distributed data processing on the Pakistan Energy Access Household Panel Survey dataset. Data Engineering

Data Quality Rules

All analyses are designed to produce clean, null-free, meaningful results — applied consistently across every visualization and aggregation.

🔤
String Normalization
All string categories are trimmed of whitespace and filtered to remove empty strings and the literal "UNKNOWN" label before any grouping or plotting.
🔗
Connection Status Derivation
Connection status is derived from the source field. "Unknown/NA" values are excluded from all grouped demographic analyses to ensure accurate percentages.
🔢
Safe Numeric Casting
Numeric columns (spend, distance, household size) are safely cast and non-numeric or null values are dropped before histogram and scatter plot generation.
✅
Cross-Validation
Spark SQL aggregation results are cross-validated against PySpark DataFrame API results to confirm correctness before exporting any output.
📤
JPG-Only Exports
All visualizations are saved as JPG only (no PNG output) for consistent file format and smaller storage footprint across the etl_output/plots/jpg/ directory.
⚙️
Reproducible Execution
The notebook can be executed end-to-end via nbconvert from the terminal for fully reproducible results without manual cell-by-cell execution.
